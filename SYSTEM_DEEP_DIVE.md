# Project Yolo: System Architecture & Capability Analysis

## 1. Executive Summary
Project Yolo is a state-of-the-art, highly autonomous AI orchestrator designed to operate as an expert software engineer and system controller. It leverages a decoupled LLM architecture to provide end-to-end solutions for complex engineering, research, and desktop automation tasks.

## 2. System Architecture
The system is built on a modular foundation, allowing for flexibility in LLM selection and interface interaction.

### 2.1 Core Cognitive Engine (`agent.py`)
The heart of Project Yolo is its autonomous loop.
- **Dynamic Prompting**: Utilizes a unified prompt architecture with profile-based rendering (verbose vs. compact).
- **History Management**: Implements smart history compaction via `session.py` to maintain context without overflowing token limits.
- **Cognitive Modes**:
    - **YOLO Mode**: Full autonomy for rapid execution.
    - **Safe Mode**: Human-in-the-loop (HITL) for sensitive operations.
    - **Think Mode**: Enhanced reasoning for multi-step complex plans.
- **Self-Evolution**: Integrated self-upgrade protocols that allow the agent to research, implement, and validate new capabilities within its own codebase.

### 2.2 LLM Routing & Interoperability (`llm_router.py`)
Yolo is provider-agnostic, supporting:
- OpenAI (GPT-4o series)
- Anthropic (Claude 3.5 Sonnet)
- OpenRouter (for diverse model access)
- Local/Custom Proxy endpoints.

### 2.3 Desktop & GUI Interaction
The project features a sophisticated desktop integration layer:
- **Electron App (`desktop/`)**: A modern UI for interacting with the agent, featuring real-time status bars, worker management, and markdown-rendered chat.
- **API Bridge (`api_bridge.py`)**: Connects the Electron frontend with the Python backend via a REST/WebSocket-like interface.
- **GUI Perception (`gui_ops.py`)**: Powered by Tesseract OCR and OpenCV, the agent can "see" the desktop, identify UI elements, and interact with them using Set-of-Mark (SoM) grounding.

## 3. Advanced Features

### 3.1 Autonomous Worker System
Yolo can spawn specialized background workers (`spawn_worker`) to handle sub-tasks in parallel. These workers:
- Operate in isolated contexts.
- Have specific objectives and task IDs.
- Report completion or request help from the main agent.
- Are monitored via the Desktop Workers Panel.

### 3.2 Long-Term Memory & Experience
- **Memories**: Persistent user context managed via `mem0` or similar services.
- **Experiences**: A technical "lesson learned" database (`experience_ops.py`) that records bug fixes and architectural decisions to avoid repeating past mistakes.
- **Proactive Memory**: Automatic extraction of facts and preferences during conversation.

### 3.3 Stealth Browsing
Utilizes `camoufox` and advanced web tools to perform research without being blocked by anti-bot measures, supporting pagination, DOM interaction, and link extraction.

## 4. Tool Suite
The agent has access to an extensive library of tools located in the `tools/` directory:
- **File System**: `write_file`, `edit_file`, `grep_search`, etc.
- **System**: `run_bash`, `gui_ops`, `background_ops`.
- **Knowledge**: `web_search`, `browse_url`, `memory_ops`.
- **Project**: `artifact_ops`, `mission_ops`, `experience_ops`.
