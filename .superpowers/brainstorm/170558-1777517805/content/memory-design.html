TieredMemoryEngine: Design Overview

Replacing flat vector search with a 4-tier cognitive architecture

SQLite Schema (Table-per-Tier)
  • L1_working_memory
    user_id, key, value, created_at
  • L2_episodic_memory
    id, user_id, event, importance, created_at
  • L3_semantic_memory (FTS5 enabled)
    id, user_id, fact, category, importance, created_at, updated_at
  • L4_pattern_memory
    id, user_id, pattern, confidence, created_at, updated_at
Consolidation Flow (Inline Sync)
  1. Turn Ends: Agent emits memory events.
  2. Scoring: Hybrid heuristic assigns importance (0-10).
  3. L2 Insertion: Event stored in L2_episodic_memory.
  4. Threshold Check: If count(L2) > 20, trigger sync consolidation.
  5. Consolidation: L2 events analyzed. High-importance promoted to L3_semantic. Patterns inferred to L4_pattern. Old L2 events cleared.
