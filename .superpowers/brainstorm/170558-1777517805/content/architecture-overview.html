Project Yolo: High-Level Architecture

System decomposition into Core, Gateways, and Workers

Architectural Map

The Brain (Core)

  • agent.py: Orchestrator
  • llm_router.py: Multi-LLM logic
  • prompt_builder.py: Context & Memory
  • tool_dispatcher.py: Tool execution
↔

The Senses (Gateways)

  • tui.py: Interactive terminal
  • bot.py / discord: Chat platforms
  • Electron App: Visual desktop UI
  • API Bridge: HTTP/SSE bridge

Knowledge & State

  • session.py: Live conversation state
  • yolo_v2.db: SQLite persistence
  • mem0: Long-term memory
  • Qdrant: Vector DB for RAG
↔

The Muscles (Workers)

  • worker.py: Autonomous tasks
  • yolo-core (Rust): High-performance
  • monitoring.py: System health
  • tools/: 20+ specialized tools

Key Discovery: Functional Gateway Pattern

Instead of complex inheritance, Project Yolo uses a consistent functional contract: all gateways call run_agent_turn and process a stream of TUIMessage packets via a signal_handler. This makes it incredibly easy to add new interfaces.
